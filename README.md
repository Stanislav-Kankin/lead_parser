# Telegram Signals Web App

Веб-приложение для поиска и ручной квалификации Telegram-сигналов под cold outreach.

Главная задача проекта: находить сообщения от селлеров, брендов и производителей, где есть живая бизнес-боль вокруг маркетплейсов, direct-канала, внешнего трафика, экономики роста или поиска следующего безопасного шага.

## Что делает приложение

- ищет релевантные Telegram-чаты по профилям поиска;
- собирает свежие сообщения и контекст ветки обсуждения;
- классифицирует сигнал по ICP, боли, намерению, контактности и пригодности для outreach;
- показывает рабочую базу в веб-интерфейсе;
- помогает размечать лиды как `ОК` / `Не ОК`;
- сохраняет причину `Не ОК`, чтобы улучшать фильтры и классификатор;
- генерирует черновик первого сообщения;
- ведет простой CRM-статус: прочитал, написал, ответил, теплый, встреча, продажа, архив;
- экспортирует базу в Excel.

## Запуск локально

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Открыть:

```text
http://localhost:8000/telegram-signals
```

## Запуск в Docker

```bash
docker compose up --build -d
```

В проде база лежит в примонтированной папке:

```text
./data/leads.db
```

Если нужен чистый старт:

```bash
rm -f ./data/leads.db
docker compose up --build -d
```

## Переменные окружения

```text
TELEGRAM_API_ID=
TELEGRAM_API_HASH=
TELEGRAM_SESSION_NAME=lead_signal_session
DATABASE_URL=sqlite:///./leads.db
```

В Docker используется:

```text
DATABASE_URL=sqlite:////app/data/leads.db
```

## Текущий фокус

Проект больше не использует Telegram-бота. Основной продуктовый интерфейс - веб-приложение.

Ближайшие улучшения:

- усилить разметку причин `Не ОК`;
- собрать эталонный датасет из ручной проверки;
- добавить аналитику качества источников;
- разделить рабочую базу, сырье и рыночный контекст;
- постепенно переносить правила классификации из “ключевых слов” в понятную модель ICP/CJM.
